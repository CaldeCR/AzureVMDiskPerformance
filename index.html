Key Principle: Effective disk performance is the minimum of the VM-level limit and the disk-level limit. A disk can never exceed what the VM allows, and the VM can never push more IO than the disk supports.

IOPS (Input/Output Operations Per Second)

The number of read/write operations a disk or VM can handle per second. Higher IOPS = better for transactional workloads (databases, OLTP).

Learn more about IOPS ↗

Throughput (MB/s)

The volume of data transferred per second. Higher throughput = better for sequential workloads (backups, large file copies, data warehousing).

Learn more about Throughput ↗

VM-Level Capping

Each VM size has maximum cached and uncached disk IOPS and throughput. Cached limits apply to disks with host caching enabled (ReadOnly/ReadWrite). Uncached limits apply to disks with host caching set to None.

Host Caching

ReadOnly/ReadWrite = IO counts against the VM's cached limits. None = IO counts against the VM's uncached limits. This is the critical factor that determines which VM limit applies.

Learn more about Host Caching ↗

Effective Performance

Total disk IOPS/MBps for each caching group must not exceed the VM limit for that group:

Cached Total ≤ VM Cached Limit
Uncached Total ≤ VM Uncached Limit

Common Misconfiguration

Attaching multiple high-performance disks (e.g., P30 with 5,000 IOPS each) to a VM that cannot handle the aggregate. The VM throttles all disk IO, causing latency spikes and poor performance.

1. Match VM to Aggregate Disk IO

Sum the IOPS and throughput of all attached disks (grouped by caching). Choose a VM size whose cached and uncached limits exceed those sums.

2. Use Host Caching Wisely

ReadOnly for read-heavy data disks. None for write-heavy or latency-sensitive workloads and Ultra/PremiumV2 disks. ReadWrite only for OS disks.

3. Monitor Throttling Metrics

Use Azure Monitor: VM Cached IOPS Consumed %, VM Uncached IOPS Consumed %, Data Disk IOPS Consumed %, OS Disk Queue Depth.

4. Right-Size Disk SKUs

Don't over-provision disk capacity just for IOPS. Consider Premium SSD v2 for granular IOPS/throughput tuning without paying for unused capacity.

5. Stripe for Performance

Use Storage Spaces (Windows) or LVM/mdadm (Linux) to stripe multiple disks. Aggregate IOPS = sum of individual disks (up to VM cap).

6. Understand Caching Impact

Disks with ReadOnly/ReadWrite caching use the VM's cached limit (often higher). Disks with None use the uncached limit. Misconfiguring caching can waste available performance.

Tip: Run az vm show --resource-group <rg> --name <vm> --query storageProfile to audit current disk configuration and caching settings.
